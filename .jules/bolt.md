## 2024-04-19 - Optimizing Python loop overhead and allocations
**Learning:** When trying to optimize list operations like slicing (`features[:10]`), replacing them with `islice()` or manual `for` loops isn't always the right move. The overhead of a manual `for` loop with conditionals `if i < 10` inside Python can actually be significantly slower than Python's native C-level implementation of `[:10]`. In addition, extracting inline lists (like `['ai', 'ml']`) from generator comprehensions inside loops into module-level constants avoids redundant instantiation on every loop iteration, providing a measurable speedup. Using `sys.modules` overhead to dynamically import things for optimization actually hinders it.
**Action:** When extracting hardcoded keyword lists, move them to module-level constants. Use `itertools.islice()` over list slicing ONLY if memory is truly constrained, otherwise native list slicing is well-optimized. Do not sacrifice code readability for theoretical optimizations. Avoid importing standard libraries locally inside tight loops unless strictly necessary for startup times.
