💡 What:
1. Replaced an N+1 query loop in `manage_marketing_campaigns` in `level6_agent.py` with a single `LEFT OUTER JOIN` + `GROUP BY` database query.
2. Refactored a sequential `any()` keyword lookup loop inside `_map_task_to_domain` in `expert_integration.py` to use a short-circuiting dictionary mapping search.

🎯 Why:
1. Fetching all businesses and then looping over them to count their `MarketingCampaign`s results in N+1 database roundtrips, degrading performance significantly as the number of businesses scales.
2. Searching a block of text using `any()` over a dynamically allocated list 8 separate times implies an `O(N*M)` execution block per domain lookup. A single pre-compiled lookup dictionary optimizes this.

📊 Impact:
- Replaces O(N) database queries with a single O(1) query for fetching businesses lacking campaigns.
- Dramatically speeds up task-to-domain mapping by short-circuiting nested dictionary iteration instead of generating and tearing down generators.

🔬 Measurement:
- Verify the marketing database query by monitoring database connections. Verify the domain mapping execution using local mock tests. Both have been validated functionally.
