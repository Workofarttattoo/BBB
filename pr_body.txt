🎯 **What:** Adds unit tests for the asynchronous `process_payment` API endpoint in `api_features.py` and implements the `process_charge` method on `PaymentProcessor`.
📊 **Coverage:** Covers the happy path (successful charge, returns status="succeeded") and the failure path (failed charge or Stripe error, returns status="failed") by fully mocking the Stripe dependency.
✨ **Result:** A critical financial path is now covered by deterministic unit tests without making actual external HTTP requests, allowing for safe refactoring of payment integration code.
