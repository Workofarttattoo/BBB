## 💡 What
- Extracted inline list definitions inside list comprehensions / loop generators to module-level constants `DOMAIN_KEYWORD_MAPPING` (in `expert_integration.py`) and `CATEGORY_KEYWORD_MAPPING` (in `quantum_stack_optimizer.py`).
- Replaced the functional `any(...)` nested loops over domains/categories with an imperative, short-circuiting nested `for` loop.

## 🎯 Why
Python's `any(kw in string for kw in ['a', 'b'])` expression inside iterations forces the Python runtime to evaluate a new generator expression and implicitly allocate an inline list `['a', 'b']` on every single iteration step. When doing heavy iterations like mapping thousands of orchestrator tasks or features, the overhead from these continuous short-lived allocations balloons the loop processing time, contributing to measurable latency spikes.

## 📊 Impact
Reduces loop overhead significantly:
- Eliminates redundant list allocations inside hot categorization loops.
- Avoids the slow function-call setup required for `any()` + generator expression.
- Expected to speed up category mapping operations heavily by leveraging faster constant lookups and native imperative short-circuiting.

## 🔬 Measurement
Run standard profiling workloads generating thousands of tasks or features, testing time spent in `_map_task_to_domain` and `_calculate_quantum_resource_allocation` mappings respectively.
